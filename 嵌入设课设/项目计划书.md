## 一、项目概述

本项目基于 **昇腾310B（OrangePi AIpro）** 开发板，利用 **MobileNet-SSD** 轻量级检测模型在 NPU 上进行实时目标检测，结合 **DeepSORT** 跟踪算法（卡尔曼滤波 + 匈牙利算法）实现多目标轨迹追踪。检测与跟踪结果通过串口发送至 **STM32F103C8T6**，驱动二轴舵机云台随目标转动，使目标始终保持在画面中心区域。

## 五、技术方案概要

### 5.1 系统架构

```
┌─────────────────────────────────────────────────┐
│                  OrangePi AIpro                  │
│  ┌──────────┐   ┌──────────┐   ┌─────────────┐  │
│  │ USB Camera│──▶│MobileNet │──▶│  DeepSORT   │  │
│  │ (视频流)  │   │ -SSD NPU │   │  跟踪器     │  │
│  └──────────┘   │  推理    │   │ (卡尔曼+    │  │
│                 └──────────┘   │  匈牙利)    │  │
│                               └──────┬──────┘  │
│                                      │ 串口     │
│                                      ▼  (目标坐标)│
└──────────────────────────────────┬──────────────┘
                                   │ UART-TTL
┌──────────────────────────────────▼──────────────┐
│                 STM32F103C8T6                    │
│  ┌────────────┐   ┌──────────┐   ┌───────────┐  │
│  │ 串口接收   │──▶│ PID计算  │──▶│ PWM输出   │  │
│  │ (目标坐标) │   │ (偏差→   │   │ (舵机驱动)│  │
│  └────────────┘   │  控制量) │   └─────┬─────┘  │
│                   └──────────┘         │         │
└────────────────────────────────────────┼─────────┘
                                         ▼
                               ┌─────────────────┐
                               │  二轴舵机云台    │
                               │  (SG90 ×2)      │
                               │  水平 + 俯仰    │
                               └─────────────────┘
```

### 5.2 软件栈

| 组件 | 技术选型 |
|------|---------|
| AI推理框架 | CANN 7.0+ + PyACL |
| 目标检测模型 | MobileNet-SSD（OM格式） |
| 目标跟踪算法 | DeepSORT（卡尔曼滤波 + 匈牙利匹配） |
| 编程语言 | Python 3.9+（昇腾端）、C（STM32端） |
| 计算机视觉 | OpenCV 4.8+ |
| 科学计算 | NumPy、SciPy |
| 串口通信 | Python `pyserial` / STM32 USART |
| MCU开发 | Keil MDK / STM32CubeIDE + HAL库 |
| 版本管理 | Git + GitHub |

### 5.3 设计步骤（详细）

1. **环境搭建**（成员A）
   - OrangePi AIpro 烧录系统、配置 CANN 7.0+
   - 创建 Python 虚拟环境，安装 OpenCV、NumPy、SciPy、pyserial
   - STM32 开发环境（Keil/STM32CubeIDE）搭建
   - 验证 NPU 设备可正常调用

2. **模型准备**（成员B）
   - 从开源仓库或参考链接下载 MobileNet-SSD 的 ONNX 模型
   - 使用 ATC 工具转换为 OM 格式（可同时准备 CPU 模式作为回退方案）
   - 编写模型加载与推理测试脚本，验证 NPU 推理正确性

3. **检测与跟踪调试**（成员A + 成员B）
   - 运行 `detection_app.py`：理解预处理（resize、normalize）→ NPU推理 → 解码 → NMS 全流程
   - 运行 `tracking_app.py`：理解卡尔曼滤波预测、IOU 矩阵计算、匈牙利匹配、轨迹管理
   - 调优检测置信度阈值、IOU 阈值、跟踪最大丢失帧数等关键参数

4. **STM32云台开发**（成员C）
   - 定义串口通信协议（帧头+目标坐标x,y+帧尾，波特率115200）
   - SG90 舵机 PWM 驱动：TIM 产生 50Hz PWM，脉宽 0.5ms~2.5ms 对应 0°~180°
   - 二轴 PID 控制：以画面中心为期望位置，目标坐标偏差为误差输入，输出 PWM 控制量
   - 串口中断接收昇腾310B发送的坐标数据

5. **系统集成与测试**（全体）
   - 联调检测→跟踪→串口→云台→摄像头闭环
   - 优化响应延迟（推理帧率、串口波特率、PID 参数整定）
   - 测试不同光照、不同目标下的跟踪稳定性
