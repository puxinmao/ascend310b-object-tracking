## 一、项目概述

本项目基于 **昇腾310B（OrangePi AIpro）** 开发板，利用 **MobileNet-SSD** 轻量级检测模型在 NPU 上进行实时目标检测，结合 **DeepSORT** 跟踪算法（卡尔曼滤波 + 匈牙利算法）实现多目标轨迹追踪。检测与跟踪结果通过串口发送至 **STM32F103C8T6**，驱动二轴舵机云台随目标转动，使目标始终保持在画面中心区域。

本项目为**二次开发**性质——开发环境（CANN 8.0）已由小组成员搭建完成，基础代码（检测推理与跟踪框架）已由老师提供并烧录至开发板。小组成员在此基础上的核心工作为：**消化理解代码 → 调试调优 → 集成云台控制 → 全链路联调**。

## 五、技术方案概要

```
┌─────────────────────────────────────────────────┐
│                  OrangePi AIpro                  │
│  ┌──────────┐   ┌──────────┐   ┌─────────────┐  │
│  │ USB Camera│──▶│MobileNet │──▶│  DeepSORT   │  │
│  │ (视频流)  │   │ -SSD NPU │   │  跟踪器     │  │
│  └──────────┘   │  推理    │   │ (卡尔曼+    │  │
│                 └──────────┘   │  匈牙利)    │  │
│                               └──────┬──────┘  │
│                                      │ 串口     │
│                                      ▼  (目标坐标)│
└──────────────────────────────────┬──────────────┘
                                   │ UART-TTL
┌──────────────────────────────────▼──────────────┐
│                 STM32F103C8T6                    │
│  ┌────────────┐   ┌──────────┐   ┌───────────┐  │
│  │ 串口接收   │──▶│ PID计算  │──▶│ PWM输出   │  │
│  │ (目标坐标) │   │ (偏差→   │   │ (舵机驱动)│  │

                                         ▼
```

### 5.3 二次开发工作内容

#### 第一阶段：代码理解（吴铭帆、梁乐天）

1. **理解检测流程**：阅读 `detection_app.py`，梳理图像预处理（resize、normalize）→ 加载OM模型 → NPU推理 → 结果解码 → NMS 全流程，标注关键参数
2. **理解跟踪流程**：阅读 `tracking_app.py`，梳理卡尔曼滤波预测、IOU 矩阵计算、匈牙利算法匹配、轨迹创建/更新/删除逻辑
3. **理解串口通信**：阅读昇腾端串口发送代码，确认与 STM32 的通信协议（帧格式、波特率、数据位等）
4. **运行验证**：在现有环境上运行程序，观察检测与跟踪效果，记录当前表现

#### 第二阶段：参数调优（吴铭帆、梁乐天）

1. **检测参数调优**：调整置信度阈值、NMS 的 IOU 阈值，测试不同光照/距离下的检测效果
2. **跟踪参数调优**：调整卡尔曼滤波的过程噪声/测量噪声协方差、最大丢失帧数 (`max_age`)、匹配阈值
3. **性能基准测试**：记录当前帧率、跟踪ID切换次数，作为后续优化的基准

#### 第三阶段：STM32云台控制开发（张泽滨）

1. **硬件连接**：
   - SG90 舵机信号线 → STM32 TIM PWM 输出引脚（两路）
   - ST-Link → STM32 SWD 调试接口
   - 昇腾310B UART TX → STM32 USART RX（通过 USB转TTL 模块）
2. **PWM驱动**：配置 TIM 产生 50Hz PWM，脉宽 0.5ms~2.5ms 对应 0°~180°
3. **串口接收**：USART 中断接收昇腾310B发送的目标坐标 `(x, y)`
4. **PID控制**：以画面中心 `(cx, cy)` 为期望位置，目标坐标偏差为误差输入，输出 PWM 控制量，实现水平和俯仰两轴闭环控制

#### 第四阶段：系统联调（全体）

1. **协议对齐**：确认昇腾端串口发送的坐标格式与 STM32 接收解析一致
2. **全链路测试**：摄像头 → 检测 → 跟踪 → 串口 → STM32 → 舵机云台，闭环验证
3. **延迟优化**：调整推理帧率、串口波特率（115200+）、PID 参数整定，减少跟踪滞后
4. **边界测试**：目标快速移动、目标短暂离开画面后重新出现、多目标场景
