### 3.3 调试中遇到的问题与解决措施

#### 问题 1：串口接收响应慢（手动调试时）

- **现象**：PC 串口助手发送数据，舵机等待数秒才开始响应
- **原因**：回调函数中直接读 USART DR 寄存器，HAL 库在进入回调前已读走数据，导致读到无效值
- **解决措施**：改用固定缓存 `rx_byte` 配合 `HAL_UART_Receive_IT()`，回调中从缓存读取，响应恢复正常

#### 问题 2：与昇腾联调时舵机来回摆动

- **现象**：人物在画面最左侧时，舵机向右转→人物回到中央→昇腾发 0x5A→控制逻辑误认为"目标 90°"又重新回位→来回振荡
- **原因**：测试模式下使用"串口值 = 舵机角度"的直接映射，而昇腾发来的是"人物在画面中的位置"，两者含义不同
- **解决措施**：切换为跟踪模式（误差控制），计算 `误差 = 人物位置 - 画面中央`，误差为 0 时舵机保持不动

#### 问题 3：昇腾与 STM32 坐标系不一致

- **现象**：人物在画面左侧时舵机反而向右转，跟踪方向完全相反
- **原因**：昇腾端的坐标系定义与 STM32 端预期方向不一致
- **解决措施**：在 `Control_Compute` 中将误差取反，匹配昇腾坐标系方向（`adjustment = -KP * error`），跟踪方向恢复正常

#### 问题 4：垂直方向跟踪参数独立调节

- **现象**：增加垂直舵机后，垂直方向跟踪响应与水平方向不一致
- **原因**：垂直方向受重力影响，所需控制参数与水平方向不同
- **解决措施**：为垂直方向独立配置 KP、死区和限步参数，两轴分开调参

### 4.3 核心控制算法

```c
/* 误差 = 人物位置 - 画面中央(90°) */
float error = target_angle - SERVO_CENTER_ANGLE;

/* 死区：误差小于 2° 不动作，防抖动 */
if (fabsf(error) > DEADBAND_ANGLE)
{
    float adjustment = KP * error;                  /* 比例控制 */
    if (adjustment >  MAX_STEP_ANGLE)  adjustment =  MAX_STEP_ANGLE;  /* 限幅 */
    if (adjustment < -MAX_STEP_ANGLE)  adjustment = -MAX_STEP_ANGLE;

    float new_angle = current_angle + adjustment;   /* 更新角度 */
    uint16_t new_pulse = 500 + new_angle * 2000/180; /* 角度 → 脉宽 */
    Servo_SetPulse(new_pulse);
}
/* 误差 < 2°：舵机保持不动 → 人物在画面中央 → 系统稳定 */
```

## 五、存在的问题与风险

### 5.1 当前问题

| 问题 | 描述 | 影响 | 应对方案 |
|:----|:----|:----|:---------|
| 水平/垂直参数未独立标定 | 两轴物理特性不同，共用参数影响效果 | 垂直方向响应可进一步优化 | 分别为 Pan 和 Tilt 配置独立的 KP/死区/限步参数 |
| 舵机转速限制 | 普通舵机转速约 60°/0.12s | 人物快速移动时可能有滞后 | 可调整控制参数或更换高速舵机 |

### 5.2 潜在风险

| 风险项 | 可能性 | 影响程度 | 预防/应对措施 |
|:------|:------:|:--------:|:-------------|
| 光线变化影响人物检测 | 中 | 中 | 加强昇腾端检测算法的鲁棒性 |
| 串口传输受到干扰 | 低 | 高 | 使用屏蔽线、缩短传输距离、添加数据校验 |
| 舵机长时间运行过热 | 低 | 低 | 避免长时间连续高强度跟踪，适当休息 |
| 电源供电不稳定 | 低 | 中 | 舵机和 STM32 分别供电，做好滤波 |
