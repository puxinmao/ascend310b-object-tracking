## 2. 系统方案设计

### 2.1 系统总体架构

本系统实现 **基于视觉的目标跟踪**，整体采用 **昇腾 310B + STM32F103** 的主从双处理器架构：

```
 ┌───────────────────────────────────────────────────────────┐
 │                    系统总体架构                            │
 │                                                           │
 │  ┌────────────────────────┐   UART    ┌───────────────┐  │
 │  │    昇腾 310B 开发板     │ ←───────→ │ STM32F103C8T6 │  │
 │  │  (读取人物位置串口信息) │  串口通信  │  (控制舵机)    │  │
 │  │                        │           │               │  │
 │  │     ┌──────────┐       │           │   ┌────────┐  │  │
 │  │     │  摄像头   │       │           │   │  舵机  │  │  │
 │  │     └──────────┘       │           │   └────────┘  │  │
 │  └────────────────────────┘           └───────────────┘  │
 │                                                           │
 └───────────────────────────────────────────────────────────┘
```

**系统工作方式说明**（基于已有信息）：

1. 昇腾 310B 开发板连接摄像头，不断读取视频画面中人物的串口信息（位置数据）
2. 昇腾 310B 通过串口将人物位置数据发送给 STM32F103C8T6
3. STM32 根据接收到的位置信息控制舵机转动
4. **控制目标**：使捕捉到的人物的串口值始终保持不变，即让人物始终位于摄像头的画面中央

> **注**：人物检测的具体算法（如 AI 模型类型、传统视觉方法等）及串口通信的协议格式，需由项目实际实现确定，此处暂不假定。

#### 3.2.2 关键算法描述

> **待补充**：以下算法需根据实际项目需求和技术选型确定。

**（1）人物检测算法（昇腾 310B 端）**

- 具体采用何种检测方法，需根据昇腾 310B 开发板上实际可用的框架和模型确定

#### 3.2.4 任务调度设计（FreeRTOS 任务划分）

> **说明**：STM32F103C8T6 上是否使用 FreeRTOS，需根据项目实际需求确定。如果系统逻辑简单（如仅在主循环中轮询串口接收 + 更新舵机），则可以不使用操作系统。

**如果采用 FreeRTOS，建议任务划分如下（待细化）**：

| 任务名称 | 优先级 | 功能描述 | 说明 |
|:---|:---:|:---|:---|
| UART 数据处理任务 | 最高 | 解析串口接收数据，提取人物位置信息 | 可由中断+信号量触发 |
| 舵机控制任务 | 中 | 根据位置数据计算目标角度，更新 PWM | 可由消息队列传递角度值 |
| 状态指示任务（可选） | 低 | 指示灯、调试信息输出等 | 辅助功能 |

> **注**：以上任务划分仅为建议，实际是否需要多任务、任务间如何通信，需根据系统复杂度确定。
